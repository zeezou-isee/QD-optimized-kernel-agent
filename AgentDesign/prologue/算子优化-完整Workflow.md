# 算子优化完整 Workflow:宏微观解耦的 QD 搜索体系

> 本文档给出**单算子 + 单硬件**下、以"**尽可能少的真机实测轮数得到更优 kernel**"为目标的端到端流程。
> 适用范围:**双后端通用骨架**(CPU/ARM + Vulkan 移动 GPU),后端相关部分(BD 第三轴、约束方程、参数化机制、硬件查询)做成可插拔小节。

---

## 0. 一句话主线

> 用 **roofline 先验**把问题分流到 memory-bound / compute-bound 两套坐标系之一;
> 用 **MAP-Elites(外层,LLM 当变异/重组算子)** 在该坐标系上照亮一张"行为 niche → 最优 kernel"档案;
> 每个候选 kernel 先过 **正确性 oracle**(对拍框架原生实现),再交 **内层参数搜索(解析剪枝 + 粗网格 + 局部爬山)** 求其 basin 实测延迟;
> 用 **质量偏置采样 + roofline 逼近即停 + 解析/经验预筛** 把总实测压在 **50–150 次**;最终只报档案 argmin。

---

## 1. 设计目标与对账(为什么是这套)

| 目标 | 设计响应 |
|---|---|
| **少轮数**(总实测 50–150) | 粗离散档案 + 质量偏置 emitter + roofline 逼近即停 + 解析/经验预筛;QD 当**手段**不当目的,只报 argmin |
| **更优 kernel**(抗欺骗) | MAP-Elites 局部竞争 → 不同算法族/布局互不淘汰,踏脚石跨越负交互山谷 |
| **正确性**(新增硬闸) | 每个候选过对拍 oracle,**先验正确再谈快慢**;错误 kernel 不得进档案 |
| **可复用**(兵器谱) | BD 坐标系按**瓶颈类型**收敛成两套(非每算子一套),同 regime 内可热启动迁移 |

> 关键取舍:目标从主文档的"零成本实测"改回"**少轮数**"后,QD 的立论从"评估付得起"转为"**抗欺骗鲁棒性**"。
> 因此本流程**必须配一个轻量对照**(纯解析/BO 引导的 best-first),用 best-fitness 判断 QD 多花的轮数是否值回票价。详见 §9。

## 2. 矛盾消解清单(相对旧两份文档)

本文档对旧方案做了如下**明确修正**,后续各节落实:

1. **BD 不再 per-operator 动态生成** → 收敛为**两套 bottleneck-conditional 坐标系**,且第二轴与延迟正交(消除 fitness 共线 + 档案不可迁移)。详见 §4。
2. **补正确性 oracle** → 旧方案只查 `compile_success`,会把"能跑但算错"的 kernel 选成精英。新增对拍闸。详见 §5。
3. **补测量 harness 与噪声处理** → latency 不再当干净标量;定义多次测量、取值规则、控温控频。详见 §5。
4. **内层 TPE → 解析剪枝 + 粗网格 + 局部爬山** → 5–10 轮预算下 TPE 仍在随机启动期,名不副实。详见 §6。
5. **"低保真度=x86 主机跑"降级为正确性闸** → x86 延迟与 ARM/GPU 不同序,不能当延迟 fidelity;缩张量会在悬崖处翻转排序。详见 §5。
6. **"正交交叉=稳赚加法"修正为"去对应格子竞争"** → 负交互可能使 (2,2) 比父代更慢,赢不赢由实测定。详见 §7。
7. **失败封锁粒度细化** → 参数级失败(OOM)不上升为封锁整个 BD 区域,只回喂内层约束。详见 §5/§7。
8. **补 roofline 早停** → 旧方案只有"30 代封顶"固定预算;新增"逼近天花板即停"。详见 §7/§8。

## 3. 三角色与数据流契约

沿用主文档的 Proposer / Evaluator / Policy 三角色,本文档给出**可实现的接口**:

| 角色 | 由谁担任 | 输入 | 输出 |
|---|---|---|---|
| **Proposer** | LLM(结构层)+ 硬件约束方程(参数层) | 父代精英 + roofline 余量 + 经验池命中 | 带 `<PARAM>` 占位符的**参数化模板** + 候选参数值 + "为何应有增益"的理由 |
| **Evaluator** | **正确性 oracle + 真机测量 harness** | 具体化的 kernel 源码 | `{correct: bool, latency: float, profile: {...}}`(真值) |
| **Policy** | 外层 MAP-Elites + 内层参数搜索 | 档案 + roofline 先验 | 展开哪个精英、何时停;最终 argmin kernel |

**核心数据流(一个候选的生命周期):**

```
父代精英 ──Proposer(LLM)──► 参数化模板 T(带占位符 + 候选值 + 理由)
                                    │
                          ┌─────────┴─────────┐
                          │ ① 解析预筛(免实测)│  主轴预定位(结构标签BD) + roofline 砍分支
                          └─────────┬─────────┘  + 经验池排序 + 硬件约束砍非法参数 + 跳过已满 niche
                                    │ 通过的才下放(主轴格 cell_main 此时已知)
                                    ▼
                          ② 内层参数搜索(实测)
                          粗网格 + 局部爬山,每点先过正确性 oracle
                                    │
                          basin 价值 = (best_latency, best_params, profile)
                                    ▼
                          ③ 落格 = cell_main + 后验精修轴细分(§4.3)
                                    │
                          ④ 同 cell 竞争:correct 且 latency 更低 → 替换精英
                                    ▼
                          ⑤ roofline 逼近度检查 → 触发早停?
```

**接口契约(两层解耦的关键,外层不关心内层怎么搜):**

- 外层 → 内层:`(parameterized_template, candidate_values, hardware_specs)`
- 内层 → 外层:`(best_params, best_latency, correct, profile)` —— 即该模板的"basin 价值"
- BD **不由内层返回**,而由外层**静态算**(见 §4.3),避免依赖 LLM 提取出错污染档案。

## 4. 阶段一:瓶颈诊断、先验剪枝与 BD 坐标系选择

### 4.1 算子画像与瓶颈诊断(决定走哪套坐标系)

输入:算子定义(如 `GEMM(M,N,K)` / `Conv(...)`)、框架原生参考实现、目标硬件标识。

1. **静态算术强度**:`AI = FLOP / bytes`(按 **naive 实现**估,不依赖具体优化)。
2. **roofline 定位**:用硬件的 ridge point(算力/带宽比)判 `AI` 落在 ridge 左(memory-bound)还是右(compute-bound)。
3. **regime 判定**:输出 `regime ∈ {memory_bound, compute_bound}` —— **这一步唯一作用是选 BD 坐标系**。

> 注意(主文档 5.1):**瓶颈是实现相关的、会随优化翻转**(naive conv 是 memory-bound,充分 tiling 后可能变 compute-bound)。
> 因此 **regime 在"问题级"(算子+shape+硬件,按 naive)判定并锁定坐标系**;个别 kernel 在搜索中漂过 ridge 属正常,由"算术强度公共轴"吸收(见 §4.2)。

### 4.2 bottleneck-conditional 行为描述符(BD)

> **为什么按瓶颈分两套**:roofline 决定哪类优化有意义,从而决定哪类多样性值得照亮。
> 这把坐标系从"每算子一套(N 套、不可迁移)"收敛成**恰好两套**,在**同 regime 内**保持可比、可热启动(治好旧方案的不可迁移问题)。
> **两条护栏**:① 第二轴必须与延迟正交(不用"标量→向量→向量+展开"那种优化深度阶梯,否则 fitness 共线、QD 报废);② regime 在问题级锁定。

**坐标系 A — Memory-bound(主杠杆 = 字节 / 访存模式):**

| 轴 | 取值(后端实例) | 类型 | 成本 |
|---|---|---|---|
| **轴1(锚)数据布局/访存模式族** | Vulkan: `{buffer,image}×{NCHW,NHWC,NC4HW4}`;CPU: `{NCHW,NHWC,NC4HW4 packing}` | 类别 | **结构,生成时即知** |
| **轴2 分块策略族** | `{无分块 / 单层 tiling / 双层 tiling+寄存器分块}` | 类别 | **结构,生成时即知** |
| (可选轴3)融合度 | `{无融合 / 融 1 个 / 融 ≥2}` | 类别 | 结构,免费 |
| (后验精修)工作集 cache 档位 | `{装入 L1 / 装入 L2 或 local mem / 溢出}` | 离散档 | 参数依赖(内层后由 tile 算) |

- 两根**主轴(轴1/轴2)都是结构标签**,模板一生成即知 → 满足"内层前定位"(与坐标系 B 对称,见 §4.3)。
- 轴2 用"**分块策略族**"(一个结构决策:用不用 tiling、几层),**不是** tile 的具体数值;它捕捉访存局部性的根本分叉,与延迟正交。
- "**工作集 cache 档位**"(那个"悬崖")**降级为后验精修轴**:它依赖内层定下的 tile,只能在内层后算出,用于在同一(布局×分块族)格内**细分**,**不承担内层前定位职责**。
- **算法族在此 regime 是弱轴**(降 FLOP 对 memory-bound 无效,主文档 5.1)→ 不当主轴。

**坐标系 B — Compute-bound(主杠杆 = FLOP / 吞吐):**

| 轴 | 取值(后端实例) | 类型 | 成本 |
|---|---|---|---|
| **轴1(锚)算法族** | `{direct, im2col-gemm, winograd23, winograd43, fft, dw, 1×1}` | 类别 | **结构,生成时即知** |
| **轴2 计算映射/指令类** | CPU: `{标量, NEON 向量化, dotprod/sdot}`;Vulkan: `{标量, vec4, cooperative-matrix}` | 类别 | **结构,生成时即知** |
| (后验精修)occupancy/ILP 档位 | `{低 / 中 / 高}` | 离散档 | 参数依赖(内层后由寄存器/线程估或便宜实测) |

- 算法族在此是**最重要的轴**(Winograd 等降 FLOP 算法能真赢),也是防"大策略塌缩"的关键。
- 轴2 捕捉"怎么喂满计算单元",是计算路径的根本分叉,与延迟正交。
- 两根**主轴(轴1/轴2)都是结构标签** → 满足"内层前定位"。occupancy 同 memory-bound 的工作集档位一样,**依赖参数 → 降级为后验精修轴**,不承担定位职责。

**两套坐标系对称:主轴全部由结构标签构成,内层前即可完全定位;参数依赖的量(工作集档位 / occupancy / 算术强度)统一降级为后验精修轴。**

**两套共用一根可选公共轴 —— 算术强度(按 roofline ridge 归一)**:roofline 跨 regime/跨算子可比,并在 regime 边界吸收瓶颈翻转。**注意它本质参数依赖**(tile 影响实际字节数)→ **只当后验精修轴 / 跨 regime 参考量,不承担内层前定位职责**。维度高时(主轴≥4)档案用 **CVT** 取代规则网格(主文档 A.5)。

### 4.3 BD 静态计算与"内层前定位"(不让 LLM 提取)

能静态算的轴**一律静态算**,不让 LLM "看代码猜 BD"(LLM 提取出错 → kernel 落错格 → 档案污染)。BD 轴按"何时可知"分两类:

| 轴类型 | 何时可知 | 例子 | 职责 |
|---|---|---|---|
| **结构标签轴(定位轴)** | **生成时即知**(模板自带标签) | 算法族、布局、计算映射/指令类、分块策略族、向量宽度、融合度 | **内层前定位**(决定 niche) |
| **参数依赖轴(精修轴)** | **内层定参后才精确** | 工作集 cache 档位、occupancy、算术强度 | 内层后**同格细分** |

> **推论(两套坐标系对称成立)**:两套坐标系的**主轴(轴1/轴2)全部是结构标签轴** → **内层搜索前即可完整定位 niche 网格位置**。
> 因此本流程**在内层搜索前即可按主轴预定位**,决定"这个结构 niche 要不要花预算"(支撑 §8 的少轮数预筛);
> 参数依赖的精修轴在内层结束后**用于同格细分**,不改变主轴定位、不影响预筛决策。

### 4.4 先验边界裁剪(免实测,砍死区)

由 LLM 写出**硬件物理约束方程**(不是猜参数,而是推导可行域),配合 roofline 砍结构分支:

- **结构级**(roofline 驱动):memory-bound **不生成 Winograd/FFT** 等降 FLOP 分支;compute-bound 不把带宽优化当主攻。
- **参数级**(约束方程驱动,后端实例):
  - Vulkan:`TILE_M*TILE_N*elem_bytes ≤ maxComputeSharedMemorySize`;`wg_size ≤ maxComputeWorkGroupInvocations`。
  - CPU/ARM:`unroll ≤ 8`(防寄存器溢出);工作集 `≤ L1/L2` 容量;tile 须整除 / 对齐到 NEON 向量宽度。
- **经验池命中**:同 regime/相似算子的历史获胜配置 → 作为先验种子与排序加权(不过滤,只加权)。

> 约束来源 = **Vulkan 运行时查询 + 微基准探针**(主文档 8.2),不依赖 NDA spec。结果即"哪些点根本不送进真机"。

## 5. 阶段二:Evaluator —— 正确性 oracle + 测量 harness(真值闸)

> 这是旧两份文档**完全缺失**的一环,也是整个系统的真值来源。**先验正确,再谈快慢。**
> 一个"能编译、能跑、还很快、但算错"的 kernel 若进了档案,会污染整条进化线 —— 必须在进档案前拦下。

### 5.1 正确性 oracle(对拍,硬闸)

每个具体化 kernel **在计时前**先验正确:

1. **参考实现 = 框架原生 kernel**(MNN/NCNN 原生 C++,你们手上已有)在同一输入上的输出。
2. **输入**:固定随机种子生成的代表性张量(覆盖典型 shape;含边界 shape 如非整除、奇数通道)。
3. **判定**:数值容差比对。
   - fp32 严格等价路径:`max_abs_err / max_rel_err` 在紧容差内。
   - fp16/int8 等**改数值结果**的路径:用**该精度的标称容差**(不与 fp32 比绝对值),精度本身当**约束轴**而非质量轴。
4. **不通过 → 直接丢弃,不计入延迟,不进档案**;错误信息回喂 Proposer(见 §7 自愈)。

> 与旧方案区别:旧方案只查 `compile_success`。编译成功 ≠ 算对。本闸把"算对"前置为进入竞争的**入场券**。

### 5.2 "x86 主机跑" = 正确性预闸,不是延迟 fidelity

旧方案把"宿主 PC 跑一遍"当低保真度延迟测试 —— **修正**:

- x86 主机延迟与 ARM/移动 GPU 延迟**几乎不同序**(微架构、SIMD 宽度、内存层级都不同),不能据此筛延迟。
- 但它**能廉价验证正确性 + 不越界 + 不 OOM** → 定位为**正确性/合法性预闸**,放在 §5.1 之前当快速过滤,**淘汰算错或崩溃的候选**,不参与任何延迟排序。
- **同理慎用多保真度缩张量**:主文档的悬崖(cache 是否装得下、对齐)本就是 size 相关的,缩小张量会改变"哪个 tile 进 cache",**在悬崖处翻转延迟排序**。本流程**只在全尺寸真机上测延迟**。

### 5.3 测量 harness 与噪声处理(latency 不是干净标量)

目标带噪(温度/DVFS,主文档 3.4),测量必须规范:

- **warmup**:丢弃前若干次(填充 cache/pipeline、触发驱动 JIT)。
- **重复 + 取值**:每配置跑 N 次(如 N=30–100),**取中位数或最小值**(最小值更接近"无干扰下限",对功耗噪声更稳健);记录方差。
- **控温控频**:测前等待降温到阈值;尽量锁频/绑大核(CPU);记录测温,异常温度的样本作废重测。
- **noise floor**:估计测量噪声幅度 `σ`;**两个精英延迟差 < σ 时视为并列**,不轻易替换(避免噪声驱动的假替换,见 §7)。

### 5.4 失败的两类处理(粒度区分)

| 失败类型 | 例子 | 处理 | 沉淀粒度 |
|---|---|---|---|
| **参数级失败** | 某 tile 触发 OOM / 寄存器溢出 | 回喂内层约束方程,缩小可行域;**不封锁结构** | 参数(内层局部) |
| **结构级失败** | 模板根本无法编译 / 语义非法 | Debugger Agent 分析,提取**全局负约束**,封锁该结构模式 | 结构(全局规则) |

> 与旧方案区别:旧方案把参数级 OOM 上升为"封锁整个 BD 网格区域",会误伤同格子里本来合法的配置。
> **修正:参数级失败只收紧内层可行域;只有结构级、不可恢复的失败才上升为全局负约束。**

## 6. 阶段三:内层参数优化(解析剪枝 + 粗网格 + 局部爬山)

> 旧方案用 TPE 但只给 5–10 轮 —— 该预算下 TPE 仍在随机启动期(Optuna 默认前 ~10 次纯随机),
> 背了贝叶斯优化的复杂度却拿不到密度模型的好处。**修正为利用"参数层局部光滑"的轻量三段式**,无冷启动问题、少预算更稳。

内层任务:给定外层下放的参数化模板,求其 **basin 价值**(局部性能天花板)。

### 6.1 三段式流程

```
输入: parameterized_template T, candidate_values, hardware_specs

① 解析剪枝(免实测)
   - 用 §4.4 的约束方程砍掉非法点(超 shared mem / 寄存器溢出 / 不整除 / 不对齐)
   - 用经验池种子值排序候选(相似算子历史好参数优先)
   → 得到合法候选网格 G(已大幅缩小)

② 粗网格实测(定位 basin)
   - 在 G 上取粗网格点(每维 2–3 档),每点:
       a. 具体化 → 正确性 oracle(§5.1)→ 不对则丢弃
       b. 测量 harness(§5.3)→ latency
   - 取最优点 p* 作为 basin 中心

③ 局部爬山 / 坐标下降(微调,利用局部光滑)
   - 从 p* 出发,沿各参数轴 ±1 档探邻域
   - 命中更优则移动,直到无改进或预算耗尽
   → 输出 basin 价值 (best_params, best_latency)
```

### 6.2 为什么这套比 TPE 适合少预算

- **参数层局部光滑**(主文档 4.1):同一机制状态内参数微调性能缓变 → 爬山天然有效,**不需要先学全局概率模型**。
- **悬崖由解析剪枝挡在网格外**(不整除/溢出在步①已删),爬山不会掉进非法区。
- **无随机启动期**:粗网格直接给 basin 定位,每一次实测都用在刀刃上。

> 适用边界:当参数维度很高(如同时有 tiling + unroll + thread + split-K 多维耦合)且预算允许时,可在步③替换为局部 BO/TPE;
> 但在 **50–150 总实测的紧预算**下,默认用爬山。该切换是**可选旋钮**,不是默认路径。

### 6.3 内层预算

- 单模板内层实测**默认 ≤ 8–12 次**(粗网格 ~4–6 + 爬山 ~4–6)。
- 这是 §8 轮数账的基本单元:外层每展开一个候选 = 一次内层预算。

### 6.4 参数化机制(后端实例)

| 后端 | 参数化方式 | 重新具体化成本 |
|---|---|---|
| **Vulkan** | **specialization constant**(tile/unroll/wg 做成 spec constant) | 近乎免费:创建 pipeline 时塞常量,驱动重优化 SPIR-V,无需重生成源码 |
| **CPU/ARM** | 模板/宏 + 重编译 | 标准 C++ 模板实例化 + 一次编译 |

> 参数化负担写进 codegen 契约,甩给 LLM(Proposer 生成模板时即把旋钮做成 spec constant / 模板参数)。

## 7. 阶段四:外层 MAP-Elites 主循环(紧预算版)

> 外层用 LLM 当变异/重组算子,在 §4.2 选定的坐标系上照亮档案。
> **紧预算改造**:粗离散档案 + 质量偏置采样 + 解析预筛 + roofline 早停。QD 当手段,最终只报 argmin。

### 7.1 冷启动:把档案铺开(质量偏置采样的前提)

> **关键澄清**:被采样的对象始终是**档案(archive)**,不是经验池。经验池只是档案的**初始填充来源**之一。
> 数据流恒为:`种子/经验池 → 填进档案 → 从档案采样`。
> 而"质量偏置采样"需要**档案里已有多个精英可供偏置**——冷启动时档案几乎是空的,所以必须先有一个"铺开"阶段,**此阶段不用质量偏置,用纯多样化**。

按可用先验分两种情形:

**情形 A:有同 regime 经验池(主文档 §8 蒸馏自 MNN/NCNN 变体菜单 + 历史获胜配置)**
- 取同 regime 的已知 kernel,直接放入对应档案格子作为**初始精英**。
- 这是"地板":**不过滤**,哪怕某种子较慢,只要占住一个空 niche 就保留(它是该 niche 当前最好)。
- 注意:经验池是**跨算子、同 regime**的,**不要求有"当前这个算子"的条目**——"compute-bound 该有哪些算法族"这类先验照样可用。

**情形 B:只有一个通过正确性检验的版本(无该算子经验池)**
这是常见起点(如框架原生 MNN kernel,或第一个 LLM 生成的正确实现)。此时档案只有 1 个精英,质量偏置无从谈起。做法:
- 把这唯一正确版本当**唯一种子 parent**。
- 用 LLM **意图驱动**地生成 N 个**风格各异**的变体——目标是**换算法族/布局/计算映射,落到不同格子把档案铺开**,而**不是求快**(铺开优先于优化)。
- 仍可借**同 regime 的通用经验池/LLM 通用知识**指导往哪些 niche 铺;真的什么先验都没有,就纯靠 LLM 通用知识 + 那一个正确版本扩散。
- 这些多样化产物各自过内层+oracle 落格 → 档案被铺到多个 niche。

**阶段切换**:当档案填充格数达到阈值(如 ≥ 主要算法族/布局轴的一半),**才从纯多样化切到质量偏置采样**(§7.2)。在此之前,采样策略 = 纯多样化(偏向空 niche / novelty),不偏 fitness。

> 一句话:**最初不从经验池"采样",而是把唯一正确版本(+ 可用的同 regime 先验)当种子,先做多样化把档案铺开;质量偏置采样在档案非空、有多个精英后才接管。**

### 7.2 主循环

```
archive = {}                       # cell -> (kernel, latency, correct)
seed_from_experience_pool(archive) # §7.1 地板播种(不过滤)

rounds = 0
while not should_stop(archive, rounds):     # 停机见 §7.4 / §8
    # ① 质量偏置选亲代(少预算:偏向高 fitness + 近 roofline + 欠探索 niche)
    parents = select_parents(archive, bias="quality + novelty + roofline_gap")

    # ② LLM 变异/重组 → 候选模板(可批量)
    candidates = LLM_vary(parents, roofline_margin, experience_pool)
        #   - 变异:对单精英语义级重构
        #   - 重组:跨"正交 BD 轴"的精英融合(见 §7.3)
        #   - 附带"为何应有增益"的理由

    # ③ 解析预筛(免实测!少预算的关键)
    candidates = analytic_prefilter(candidates)
        #   - 主轴预定位:structural_bd(T) 内层前即可算(§4.3)
        #   - roofline 砍不该走的分支(memory-bound 不要 Winograd)
        #   - 经验池 + 约束方程排序,砍明显非法/互斥(负交互)组合
        #   - 跳过"已满且难超越"的 niche(用预定位的主轴格判断)
        #   - 只放行 top-N 进入实测

    for T in candidates:
        # 主轴格已知(内层前),记为 cell_main = bin_main(structural_bd(T))
        # ④ 内层参数搜索(实测,§6)— 每点先过正确性 oracle(§5.1)
        best_params, best_lat, correct, profile = inner_param_search(T)
        if not correct:
            feedback_to_proposer(T, reason); continue        # §5.4 失败处理

        # ⑤ 落格 = 主轴格(内层前已定) + 后验精修轴细分(§4.3)
        cell = refine_cell(cell_main, posthoc_bd(best_params, profile))

        # ⑥ 同 cell 竞争(★只和同格子比★;考虑 noise floor §5.3)
        if cell is empty or best_lat < archive[cell].latency - σ:
            archive[cell] = (T@best_params, best_lat, True)

        rounds += inner_budget(T)

return argmin_latency(archive)      # QD 当手段,只报全局最快(§8.3)
```

#### 7.2.1 亲代来自档案,不是"上一轮产物"(非链式)

这是 MAP-Elites 与"链式/贪心搜索"的本质区别,容易被误解,明确写清:

- **每轮的 parent 由 `select_parents(archive, ...)` 从整个档案里挑**(主循环步①),**不是**默认接着上一轮刚生成的那个 kernel 往下走。
- 被挑中的 parent **可能是上一轮新产生的精英,也可能是好几轮前的、甚至是冷启动种子**——只要它当前占着某个 niche。

```
   最初种子 ─┐
            ├─►  档案(各 niche 的当前精英)
 上一轮产物 ─┘          │
                 select_parents 从中挑一个 ──► 这一轮的 parent ──► LLM 变异/重组
```

为什么不是两种极端:

- **不是"每轮都从最初实现重来"**:那样无累积,退化成随机重启,丢掉已发现的好中间态。
- **不是"严格沿上一轮产物链式深化"**:那是贪心,会被负交互困死(主文档 4.2:有益组合常要先过一个暂时变差的中间步)。

**好处(对应 §7.4 的"耐心/回溯")**:因为 parent 来自档案而非"上一个产物",搜索可以**随时回到任意一个有潜力的中间态去换方向**——某条路径深化到变差时,较浅的好节点仍在档案里,下一轮自然可被重新选中展开。**踏脚石不因一步变差而丢失**,这正是 best-first/回溯行为的自然来源。

### 7.3 变异/重组的三重效应(修正"正交交叉=稳赚"的表述)

LLM 对精英做语义级重构,产生三种状态跃迁(沿用旧方案的有用部分,修正其误导):

1. **跨格跃迁(BD transition)**:如在某精英上叠加优化,代码升维,落入**另一个 niche** 去竞争。
2. **同格洗腿(in-cell refinement)**:指令重排等 BD 不变的等价变换,在原格子内篡位。
3. **降级自愈(bottleneck alleviation)**:主动删减过激优化以消除 spill,代码退回较低 niche 但拿该 niche 高分。

**重组(crossover)**:在**正交 BD 轴之间**取互补精英融合(如"极佳访存"× "极致计算")。

> **修正旧表述**:旧方案把 (2,0)+(0,2)→(2,2) 讲成稳赚加法。**实际上负交互可能使 (2,2) 比父代更慢**(主文档 4.2:激进 unroll + 大 tile → spill)。
> 正确理解:重组产物**只是去 (2,2) 格子竞争**,赢不赢由实测定;若变慢,则被"降级自愈"或该格已有精英压制。
> LLM 在此的价值是**用领域知识猜"哪些组合可能协同"以减少无效尝试**,而非保证组合必然更优。

### 7.4 深度/停机:耐心 + best-first(不一退即停)

- **不**因"子节点比父节点慢"就立即剪枝(主文档 7.4:有益组合可能要先过一个变差中间步)。
- MAP-Elites 的**局部竞争本身就提供了"耐心"**:慢一点但落在空格的 kernel 存活,后续可被改进 → 踏脚石不被淘汰。
- 质量偏置采样 + novelty 注入:既偏向高 fitness 利用,又给欠探索 niche 概率,避免塌缩到已知配方。

### 7.5 与轻量基线的对照(必须做)

> 因为目标是"少轮数",QD 不是无脑首选。**并行跑一个对照**:纯解析/BO 引导的 best-first(直接 argmin,不维护档案)。
> 判优看 **best-fitness @ 相同实测预算** 与 **最优解来自哪个格子**:
> - 若 QD 的 best-fitness 更好、且最优来自非主流格子 → 多样性付费成功(landscape 欺骗性强,QD 值得)。
> - 若 best-first 在更少轮内追平 → 该算子欺骗性弱,用基线即可。
> 这把"用不用 QD"变成**数据驱动的决策**,而非信仰。

### 7.6 提议去重与停滞处理(防 agent 反复输出相似策略)

LLM 自主提议时**会**反复输出相似策略(低温时趋于"安全"方向,高温时又发散失控)。需分清两层后果:

**机制已挡住的(白吃的保护,无需额外处理):**
- 相似提议**落进同一个 cell,只有更快的能留下**,重复的被同 cell 竞争丢弃 → **不污染档案、不损害多样性**。
- 解析预筛(§7.2 步③)把重复的非法/无增益提议挡在实测之前。

**机制挡不住的(真实浪费紧预算的实测轮数):**
- "合法但相似且无增益"的提议 → 每个仍会花掉一次内层实测预算(§6.3 的 8–12 次)。在 50–150 紧预算下是实打实的损耗。

**对策(四条,按效力):**

1. **回喂"已试过什么"(最直接,OPRO 式)**:提议 prompt 中带上——该 parent 已试过哪些变异、各自落在哪个 cell、延迟多少;明确要求"提出与已尝试**不同**的方向"。
2. **novelty 定向 / 目标空格子**:不让 agent 自由发挥,而是告诉它"这些 niche 还空着 / 还很弱,往这些方向提",把提议从"自选"变成"定向填空"(也对应让 LLM 读整张档案推理)。
3. **批量提议 + 实测前去重**:一次出 N 个候选,按**结构标签 / 预测 BD** 去重,只测互不相同的;重复的在实测前合并掉,**不花轮数**。
4. **停滞检测 → 强制换 parent / 调高 novelty 权重**:若连续若干轮新候选都落进已满且打不过的格子(典型"原地打转"信号),主动切到欠探索 niche,或临时调高采样的 novelty 权重。

> 一句话:**重复提议不破坏档案的正确性与多样性(机制挡住了),但会浪费紧预算的实测轮数——靠"回喂历史 + novelty 定向 + 批量去重 + 停滞检测"四条防住。**

## 8. 轮数账、停机与输出

### 8.1 轮数账(对账"少轮数")

总实测 = 冷启动种子实测 + Σ(外层候选 × 内层预算)。紧预算目标 **50–150 次**,推荐默认:

| 项 | 默认值 | 实测开销 |
|---|---|---|
| 冷启动种子 | 4–8 个(经验池已知值,内层预算减半) | ~15–30 |
| 外层展开候选数 | ~10–15 个(解析预筛后才实测) | —— |
| 每候选内层预算 | 8–12(§6.3) | ~80–150 |
| **解析预筛淘汰率** | 越高越省(免实测砍掉无效候选) | 0 |

> 省轮的三个杠杆,按效力排序:**① 解析预筛(免实测砍候选)> ② 内层用爬山而非 TPE(每候选省一半实测)> ③ roofline 早停(提前收手)**。
> 这些都是可配旋钮;表中为起步默认,实测后按命中率调。

### 8.2 停机条件(满足任一即停)

1. **逼近 roofline**:全局最优延迟距 roofline 天花板 < ε → 已榨干可达性能,停(本领域奢侈品)。
2. **预算耗尽**:总实测达上限(如 150)。
3. **收敛**:全局最优连续 K 轮(或最近 W 步提升 < ε)无改进 → 边际递减,停。

### 8.3 输出

- **主输出**:`argmin_latency(archive)` —— 全局最快且**已验证正确**的 kernel,具体化为最终源文件(SPIR-V/C++)。
- QD 是手段:维护档案是为了抗欺骗找到更优 argmin,**不要求填满档案才停**。

### 8.4 跨任务复用(兵器谱,同 regime 内)

- 整张档案存为经验池条目:`{regime, 算子类, 硬件, niche → 最优 kernel + 参数}`。
- **复用边界**:在**同 regime** 内热启动(memory-bound conv → 另一 memory-bound conv;同后端优先)。
  跨 regime 不直接迁移格子(坐标系不同),只迁移**配置片段**作弱先验。
- 相似算子互为先验;设备 A 的获胜配置 → 设备 B 的种子(主文档 7.6)。

## 11. 仍待确认/可调旋钮

- **BD 主轴2 的具体类目**(memory-bound 分块策略族的层级划分、compute-bound 计算映射类目)需按目标设备的指令集/原语集定。
- **后验精修轴的分档界**(工作集 cache 档界、occupancy 档、AI 按 ridge 归一的 bin)需按目标设备的 Vulkan 查询/微基准结果定。
- **容差阈值**(fp16/int8 路径)需按算子下游精度需求定。
- **预算默认值**(代数、内层预算、ε、K、W)为起步推荐,跑通后按命中率回调。
- **noise floor σ** 需在目标设备上实测标定(同配置重 复测的方差)。
